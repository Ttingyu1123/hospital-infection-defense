# 醫院防疫大作戰（Hospital Infection Defense）

一款純瀏覽器、可離線執行的 2D 俯視角動作防守遊戲。你扮演一名**感染管制醫師**，
在醫院走廊阻止病原體擴散，守護位於畫面底部中央的 **ICU 重症病人**，
用三種感染控制工具擊退五波進攻，最後擊敗抗藥性 Boss。

> 本遊戲為教育與娛樂用途，不取代正式醫療建議或感染管制指引。

原創作品，未使用任何受版權保護的遊戲名稱、圖像、地圖、角色或素材；所有畫面以
Canvas 幾何圖形即時繪製，音效以 Web Audio API 即時合成。

---

## 教育目的

在動作玩法中融入感染管制核心觀念，玩家會在遊玩中自然體會：

- 抗生素**無法**治療病毒感染
- 部分芽孢對酒精**耐受性較高**，需靠紫外線／環境消毒
- 抗藥性會使治療選擇受限，需要多工具組合
- 環境污染會提高院內感染風險，需要即時清消
- 手部衛生與適當使用 PPE 可降低暴露風險

首次觸發相關情境時，畫面會跳出一次性的衛教提示卡。

## 啟動方式

不依賴任何框架、引擎、CDN 或外部檔案，兩種方式皆可：

1. **直接開啟**：用瀏覽器打開 `index.html`。
2. **本機伺服器**（建議，避免部分瀏覽器對 `file://` 的限制）：
   ```bash
   cd Game-pandemic
   python -m http.server 8791 --bind 127.0.0.1
   # 開啟 http://127.0.0.1:8791/
   ```
   > 注意：避免用 8000/8080/5173 這類熱門埠——同機其他專案的 dev server 可能已占用。

## 難度與模式

開始畫面可選：

- **難度**：簡單 / 普通 / 困難 — 調整病原體血量與速度、病人受傷倍率、玩家受傷後無敵時間。
- **模式**：
  - **劇情五波** — 五波固定關卡 + 最終 Boss，擊敗後勝利。
  - **無盡模式** — 波次程序化持續升級，每五波一隻菌王，挑戰存活與高分（無勝利，直到病人或醫師倒下）。

---

## 三種感染控制工具

| 工具 | 攻擊方式 | 剋制 | 弱項 |
|------|----------|------|------|
| ① 酒精噴霧 | 近距離扇形 AoE，同時清除前方污染 | 一般細菌、環境污染 | 對病毒普通、對芽孢**極差** |
| ② 抗生素發射器 | 直線膠囊投射物，射程長，可破壞隔板 | 一般細菌 | **對病毒無效**、對抗藥性低 |
| ③ 紫外線消毒器 | 慢速直線光束，穿透命中多個敵人並清污染 | 病毒、芽孢、污染、Boss 護盾 | 冷卻長、能量消耗高 |

每種工具有獨立的**能量池**（發射消耗、隨時間回充）與**冷卻時間**，
不同工具對不同病原體的傷害倍率不同（抗性矩陣）。

---

## 敵人類型（病原體）

| 類型 | 顏色 | 特徵 | 弱點 |
|------|------|------|------|
| 一般細菌 | 綠 | 速度中等、生命低，朝病人前進 | 酒精、抗生素 |
| 快速病毒 | 紫 | 快速、移動不規則、成群 | 紫外線、酒精（抗生素無效） |
| 芽孢細菌 | 黃褐 | 厚重、慢、高血量，會留下污染 | 紫外線、集中攻擊（酒精極差） |
| 抗藥性細菌 | 深紅 | 高血量、優先攻擊病人、快速腐蝕隔板 | 多工具組合、紫外線 |

所有病原體都會接觸腐蝕 ICU 前緣的**污染隔板與隔離門**打開通路，並在突破後持續傷害病人。

## 道具與增益

- **PPE 防護裝備**：獲得可吸收一次傷害的護盾
- **消毒補給箱**：補滿工具能量並短暫縮短冷卻
- **急救包**：回復病人生命
- **隔離警示卡**：短時間內全體病原體減速
- **洗手台**（場上固定設施）：靠近即完成手部衛生，暫時提升酒精效果、降低污染影響
- **疫苗接種站**（場上固定設施，ICU 兩側）：靠近即為病人啟動「疫苗防護」，一段時間內大幅降低病人受傷

---

## 波次說明

1. **基礎清潔** — 少量一般細菌（教移動、酒精、保護病人）
2. **病毒入侵** — 一般細菌＋病毒（教工具切換、抗生素對病毒無效）
3. **環境污染** — 加入芽孢，地圖開始出現污染區，洗手台功能吃重
4. **抗藥性危機** — 芽孢＋抗藥性細菌，隔離門可能被破壞，需組合工具與 PPE
5. **超級抗藥菌王** — 最終 Boss

每波開始有 2～3 秒提示；當前波全清才進入下一波；完成波次給分並回復病人少量生命。

---

## Boss：超級抗藥菌王（三階段）

- **第一階段**：緩慢朝 ICU 前進、定期生成一般細菌、發射污染液滴
- **第二階段（<60%）**：加速、生成病毒與芽孢、留下污染區（「抗藥性升高」）
- **第三階段（<30%）**：狂暴，攻擊頻率上升、週期性生成護盾——**用紫外線可快速破除護盾**，
  酒精清周圍污染，抗生素僅造成少量傷害

擊敗後全圖消毒、污染清除，顯示「院內感染控制成功」。

## 測試方式

需要 Node 版 Playwright（`npm i -g playwright` 且已安裝 chromium）。

```bash
# 先啟動伺服器
python -m http.server 8791 --bind 127.0.0.1

# 另開終端機
export NODE_PATH="$(npm root -g)"   # 讓 node 找到全域 playwright
node test/smoke.js     # 24 項核心功能測試
node test/features.js  # 15 項擴充功能測試（難度/無盡/疫苗/觸控）
node test/soak.js      # 60 秒穩定性
node test/shoot.js     # 截圖到 test/shots/
```

已實測結果：smoke **24/24**、features **15/15**、console 零錯誤、約 61 FPS 單一迴圈；
soak 60 秒 heap 成長約 **0MB**、物件陣列受上限控制；
波次 1→5→Boss→勝利、病人歸零→失敗、PPE 吸收、抗性教育（抗生素對病毒 0 傷害）、
難度縮放、無盡波次生成、疫苗減傷與觸控介面均驗證通過。

### 開發測試模式

遊戲中按 **`**（backtick）切換 DEBUG，開啟後：

| 鍵 | 功能 |
|----|------|
| 7 | 回復病人生命 |
| 8 | 直接完成當前波次 |
| 9 | 清除所有敵人 |
| 0 | 召喚 Boss |
| I | 切換玩家無敵 |
| H | 顯示碰撞框 |
| K | 對 Boss 造成傷害 |

DEBUG 預設關閉，不影響正式遊戲體驗。

## 已知限制

- 觸控介面提供搖桿與動作鈕，但開始畫面的難度／模式選擇目前以鍵盤為主（觸控使用預設值）。
- 敵人尋路採方向權重＋卡住偵測（非完整 A*），極少數情況下病原體會在障礙間短暫繞路。
- 污染區對玩家採「減速」而非扣血（避免以生命格計算的懲罰過重）。
